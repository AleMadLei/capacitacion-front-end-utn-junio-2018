Docker?

Problemas durante desarrollo

  • En un mismo equipo, diferencias entre los ambientes de desarrollo (Windows, Linux, Mac).
  • Para el mismo ambiente, variaciones entre las bibliotecas utilizadas.
  • Diferencias entre los ambientes finales (dev, stage, prod).
  • Dificultad para montar un ambiente completo para personas no técnicas y técnicas.

Posibles soluciones

  • Trabajar directamente en el servidor.
  • Estándarizar el equipo de trabajo para que los ambientes sean iguales al del servidor.
  • Máquinas Virtuales.
  • Docker.

Qué es?

  • Una plataforma de contenedores que permite solucionar un problema.
  • Un contenedor es un paquete ligero y ejecutable de software que incluye todo lo necesario para realizar una tarea.
  • Una analogía es como una máquina virtual muy ligera.

Esquema

Instalación

  • Windows
  • OSX
  • Ubuntu

Docker Compose

Problema

Aún así puede ser complicado inicializar un contenedor debido a los comandos desde terminal
docker run -d --name container-name -p localhost:80:80 -v $HOME/myContainer/configDir:/myImage/configDir --restart=always image-name
La complicación aumenta de acuerdo al número de contenedores que yo necesite ejecutar.

Qué es?

  • Una herramienta que permite orquestrar contenedores a través de un archivo de configuración.
  • Reduce la cantidad de comandos que un usuario debe aprender.
  • Permite mantener versionada la configuración del sistema.
